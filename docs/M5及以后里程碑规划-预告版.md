## 〇、导航与三句结论

### 0.1 先读三句

1. **M5**（~8 工作日 · ~20 人天）= **继续外网** `117.72.211.51` · 权限与账号扩容 · **50～100 人正式试点**（手册 / on-call / 数仓 SLA / LLM 配额）。  
2. **M6**（日历 **与 M5 并行**，~12～16 人天，**不受 M5 阻塞**）= **泰山内网逐步搭建** · 预发 → 双轨 → 就绪后 **用户迁回内网** · 外网再降级为维护页。  
3. **M7+** = 驾驶舱 · 压测 · 谈判教练/博弈 · 组织架构树形编辑器等 — **单独立项**，不挡 M5/M6。

### 0.2 相对旧规划的调整（2026-06-18 决策）

| 旧版（M4 预告 §三） | 新版 | 理由 |
|---------------------|------|------|
| M5 = 泰山内网一切流 | **M5 = 外网 50～100 人试点** | 内网迁移动线长、审批不可控；先让业务用起来 |
| M6+ = 50 人运营 | **M5 = 50～100 人运营** | 权限与 M4 角色矩阵已就绪，外网可承接试点 |
| M5 必须内网 URL | **M6 再切内网** | M5/M6 **双轨并行**：业务不等 infra |
| 驾驶舱/压测在 M6+ | **M7+** | 给 M5 试点 + M6 迁云让路 |

**可行性结论**：✅ **可行**。M4 已交付登录、四类角色、`user_brands`、LLM 审计/配额、数仓 bi_csv；外网具备试点条件。M6 内网建设与 M5 用户扩容 **解耦**，风险低于「先切流再推广」。

### 0.3 双轨并行示意

```text
时间轴 ─────────────────────────────────────────────────────────►

  M5 轨（外网 · 受控试点）
  ├─ 账号/权限批量配置

  ├─ DTS 预发 → 生产
  └─ 内网就绪后：用户迁移 Runbook · 外网改维护页

  交汇点：M6-D「用户迁回内网」— 日期 **以内网就绪为准**，不写入 M5 收官门槛
```

> M6 内网稳定 **+ M5/M6 试点无 P0 连续 2 周** 后开评审会。可拆子项目并行。

### 4.1 子项目 A · 驾驶舱（~8 人天）

`cockpit.html` · 团队聚合 API · P0 列表 · 拜访/承诺完成率 · manager 只读下钻。

### 4.2 子项目 B · 压测（~3 人天）

50 并发只读 API · k6/ab 报告 · Forcebot 内网全链路。

### 4.3 子项目 C · 品牌与组织扩容（~8 人天）

50+ 品牌映射 · 组织树形编辑器 · 批量档案模板。

### 4.4 子项目 D · 差异化（按需）

谈判教练 · 博弈推演 · 战局视图 · 资源争取报告 PDF 等（各 12～20 人天）。

### 4.5 推荐顺序

```text
M6 内网稳定
  → 评审
  → B（压测）与 A（驾驶舱）可并行
  → C（扩容）视业务
  → D 任选单独立项
```
