## 一、协作规则

> 与 M2 手册 §1 **相同部分不再重复**；下列为 M3 **增量**。完整铁律见 [M2 §1](M2并行开发手册-正式版.md#一协作规则)。

### 1.1 能力边界（M3）

| 能力 | M2（已交付） | **M3 新增** |
|------|-------------|-------------|
| 工程 | 规则版、baseline、合并 | **登录 Session** · **用户↔品牌** · **数仓 v1 同步** |
| LLM | 槽位 + 规则 fallback | **五路 LLM**（见 §3.1）· 开关 · 超时降级 |

**原则**：**关 LLM = M2 规则版**；LLM 失败/超时 **不算 bug**，须自动降级。

### 1.3 技术与环境（M2 延续）

栈、页面名、`?brand=`、`api-base.js`、`name_key` 五品牌 — **同 M2 §1.3**。  
**新增环境变量（Max 主工程 `.env`，模块席只读约定）**：

| 变量 | 含义 | 默认 |
|------|------|------|
| `LLM_ENABLED` | 全局 LLM 开关 | `false`（W 阶段 Max 逐路打开） |

模块席 **不要** 提交含真实 Key 的 `.env`。

| POST | `/api/dw/import/csv` | CSV 导入（admin） |
| POST | `/api/dw/sync/sample` | 样例 CSV 同步（admin / 本机 smoke） |

### B · Max 本机启动（M3）

> 路径约定：项目 `~/Desktop/sandtable` · MySQL 免安装版 `~/mysql-sandtable` · 端口 **3306 / 8000 / 5510**  
> M2 基础三步仍有效；M3 追加 **登录库初始化** 与 **登录页** 入口。

#### B.1 每次启动 — 开 **3 个终端**（各保持运行）

**终端 ① · MySQL 3306**

```bash
~/mysql-sandtable/mysql/bin/mysqld \
  --datadir=$HOME/mysql-sandtable/data \
  --basedir=$HOME/mysql-sandtable/mysql \
  --port=3306 --bind-address=127.0.0.1 \
  --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
```

等到日志出现 `ready for connections`（或另开窗口 `nc -z 127.0.0.1 3306` 通）。

**终端 ② · API 8000**

```bash
cd ~/Desktop/sandtable/server
```

#### B.7 常见问题

| 现象 | 处理 |
|------|------|
| `curl /api/brands` → Internal Server Error | 先跑 `bootstrap_local_db.py`；仍失败看终端 ② uvicorn 报错 |
| API 连接失败 / Connection refused | 顺序：① MySQL → ② uvicorn；`nc -z 127.0.0.1 3306` / `8000` |
| 页面 API 失败、Unexpected token `<` | 8000 未起，或静态页未硬刷新 |
| 登录后档案仍显示 5 个 Tab | 硬刷新；确认 `web/js/auth.js` 已加载且 `/api/brands` 带 Bearer |
| MySQL 起不来 | 勿删 `~/mysql-sandtable/data`；查 `~/mysql-sandtable` 相关日志 |
