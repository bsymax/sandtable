### 1.4 公共表与 API

**字段申请模板**（群里发，Max 批准后再动）：

```text
【字段申请】
申请人：佳璇
表：brands
新增：founded_year VARCHAR(16) …
原因：档案 Tab 需要
影响：纯新增列，不改已有列
```

**铁律**

1. `brands` / `brand_contacts`：以培翛初版为准；**只增列**，不改名/改类型/删列。
2. 交付包 `schema.sql`：**只写自己模块的表**；❌ 禁止 DROP 全库或 DROP 培翛 7 表。
3. **开开必读 breaking change**：
  - `intel_weekly_reports` **已废弃**  
  - 周报叙事 → `**brand_metrics`** 扩展列  
  - 预警 FK → `metrics_id`；`category` = `增长机会` / `风险预警`（**仅 alerts，新闻不用**）
4. 跨模块写表（如预警一键拜访）→ 先申请（一键拜访 **06-11 已确认为正式联动**）。
5. 人名种子与 `brand_contacts` 一致（苏泊尔刘明、小熊孙悦）。

**API 前缀**：拜访 `/api/visits`… · 档案 `/api/brands/profile/…` · 情报 `/api/intel/…`（全表 [附录 G](#附录-g--api-与表归属速查)）

**三步启动**

```bash
# ① MySQL 3306（后台）
~/mysql-sandtable/mysql/bin/mysqld \
  --datadir=$HOME/mysql-sandtable/data \
  --basedir=$HOME/mysql-sandtable/mysql \
  --port=3306 --bind-address=127.0.0.1 \
  --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci &

# ② API 8000
cd ~/Desktop/sandtable/server && python3 -m uvicorn main:app --host 127.0.0.1 --port 8000

# ③ 静态 5510
cd ~/Desktop/sandtable && python3 -m http.server 5510 --bind 127.0.0.1
```

**常见问题**


| 现象                   | 处理                    |
| -------------------- | --------------------- |
| API 连接失败             | 先起 MySQL，再起 8000      |
| Unexpected token '<' | 缺 `api-base.js` 或未硬刷新 |
| `/docs` 打不开          | 8000 未起               |


模块席日常开发在 **自己 baseline 包** README 启动；看合并效果用 **外网** 或只读对照包。
