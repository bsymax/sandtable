## 一、协作规则

> 与 M2/M3/M4 **相同部分不再重复**；见 [M4 §一](M4并行开发手册-正式版.md#一协作规则)。

### 1.1 能力边界（M5 增量）

| 能力 | M4（已交付） | **M5 新增** |
|------|-------------|-------------|
| 登录/权限 | 四类角色 · `user_brands` · readonly 403 | **批量开户** · **改密/重置** · **admin 用户管理** · 首次登录改密 |
| UI | 功能完整，含 M1～M4 开发痕迹 | **对外打磨**（附录 G）· 去演示/测试/mock 文案 |
| 运营 | 4 测试账号 | **50～100 试点账号** · 手册 · on-call · 巡检 |
| 数仓 | bi_csv 真数 | **周更 SLA 文档** · 连续 2 次导入成功 |
| LLM | 审计/配额 2000/60 | 按试点日活 **调配额** · 成本粗算 |
| 工程 | 外网 tag | **备份 cron + 恢复演练** · `/docs` 收敛 |
| 暴露面 | Swagger 全开 | **生产收敛**（C6） |

**原则**

1. **M5 不改业务大功能**（不建驾驶舱、不做内网切流）。  
2. **UI 改动以「试点用户看不懂内部术语」为准** — 附录 G 逐项勾选。  
3. **账号安全**：禁止在页面展示统一演示密码；禁止登录页列出测试账号。  
4. **LLM 降级仍不算 bug**；对外文案改为「智能摘要暂不可用，已显示简要说明」类表述，避免「M3 规则版」。  
5. **阻塞先 @模块牵头**，再 @Max；带截图 + URL + 账号角色。

| **佳璇** | 档案 UI · 档案试点 FAQ · **数仓 SLA** · J 回归 | ❌ git · ❌ 主工程 · ❌ 名单 |

**结论**：抽取错误时，试点只能「保存前改左侧原文」或「保存后改状态」；**缺「承诺行内编辑」** → **6/23 起 M5 补丁（培翛 UI + Max API）**，见 **K.6**。

#### K.3 核查 · 一人多品牌（**已支持，不改表结构**）

| 层 | 现况 |
|----|------|
| 库 | `user_brands(user_id, brand_id)` 多对多 |
| 导入 | CSV `brand_keys` 逗号分隔 + legacy 映射；示例含 `mgr01` 两品牌、`vp01` 五品牌 |

**结论**：开开 CSV **可以**一人多品牌；**不需要**改数据结构。事后改绑：**6/23** Max admin 页可改；或次窗重导。
