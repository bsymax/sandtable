# 品牌沙盘 · 泰山资源申请与迁移规划

> **文档性质**：Max 申请权限 + 分 M 迁移路线图

## 〇、三句结论

1. **M2 收官前**：只 **登记 + 预申请**，**不迁生产**（502/合并窗口不稳定）。  
2. **M3**：迁 **数据层 + 预发/试点应用**（云 MySQL、DTS 数仓、行云/T-PaaS、DUCC、监控）；外网 IP **可保留作 demo/应急**。  
3. **M4～M5**：迁 **网关 + 正式运维 + 全员内网**；**M5 推广前** 必须完成泰山正式链路，外网单机 **降级或下线**。

## 二、分 M 迁移策略（何时动、动什么）

### 原则

| 原则 | 说明 |
|------|------|
| **功能优先于迁移** | M2 P0 未清不做生产切流 |
| **先数据后应用** | 数仓同步依赖云 MySQL，库先迁 |
| **先内网试点后全员** | M3 内网小流量；M5 全员内网域名 |
| **外网 IP 作备份** | 117.72.211.51 保留到 M4 末，M5 前降级 |
| **申请前置** | 工单审批常 1～2 周，**比开发早半 M 申请** |

---

### M1 / M2（当前～收官）— **不迁移，只铺垫**

| 动作 | 做 / 不做 |
|------|-----------|
| ✅ 做 | **AppInfo** 登记「品牌沙盘」；拉 **数仓对接人**；填 **云 MySQL + DTS** 预申请单 |
| ✅ 做 | 评估 FastAPI **Dockerfile** + 行云 hello-world 部署（空应用验证权限） |
| ❌ 不做 | 生产库迁泰山、切 DNS、关外网 IP |
| 生产态 | 仍为 **117.72.211.51 + 单机 MySQL** |

**原因**：M2 合并/502 风险高；迁移与 feature 并行会拖 Max。

---

### M3（~8 工作日）— **第一次迁移：数据层 + 内网试点**

这是 **必须动泰山** 的里程碑（登录、数仓、50 并发、LLM 都依赖基础设施）。

#### M3-W1（D1～D4）— 申请落地 + 预发

| 迁移项 | 目标 | 验收 |
|--------|------|------|
| 云 MySQL | 建 **预发库**，导入 schema + seed | FastAPI 预发连泰山库 |
| DUCC | `LLM_ENABLED`、DB 连接等 | 改配置不重启或热更 |
| 行云 + T-PaaS | **预发** 容器跑 FastAPI + Nginx 静态 | `/docs` 200 |
| DongMonitor | 预发应用接入 | 502 有告警 |
| SSO / LLM | **方案定稿 + 配额申请**（可未切换） | 工单号有进展 |

**生产**：仍可指向 **117.72.211.51**（三人 zip 合并验收不变）。

#### M3-W2（D5～D8）— 数据正式 + 双轨运行

| 迁移项 | 目标 | 验收 |
|--------|------|------|
| **DTS + Buffalo/EasyData** | BI → **`brand_metrics` 日更** | 同步日志；档案见最新周 |
| 云 MySQL **生产库** | 从单机 **dump 迁移** 或 DTS 全量+增量 | 数据一致 smoke |
| 应用 **双轨** | ① 外网 IP 继续服务 ② **内网行云 URL** 给试点账号 | 两入口数据同源 |
| Forcebot | 对 **内网入口** 打 50 并发 | 压测报告 |
| 登录 | 临时账号 **或** SSO 若已批 | 50 测试账号 |
