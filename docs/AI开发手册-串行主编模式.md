## 使用说明（先看 3 分钟）

1. 本手册给 **全组** 用；**代码只出自主编席**，不是四人各写各的。  
2. 开发 **必须串行**：M0 → M1 → … → M10，**上一里程碑验收不过，禁止做下一个**。  
3. 卡住时统一说：**「我在 Mx 卡住了」** + 截图/报错原文。  
4. 敏感信息勿贴进公网 AI；示例用脱敏 CSV。

## 一、角色与模式说明

### 1.1 为什么叫「主编」而不是「开发」

团队 **没有人真的会写代码**。GANT 能成，靠的是 **一个人持续对 AI 下指令、验收、推进**——角色本质是 **项目主编**，不是工程师。

| 称呼 | 含义 |
|------|------|
| **主编席** | 全组 **唯一** 可让 AI 修改 `web/` 的席位（建议负责人担任） |
| **辅助席 ×3** | 验收、数据、业务规则；**不抢主编席** |
| **串行** | 同一时刻只有一个里程碑在推进，不多人同时改仓库 |

> 不推荐叫「写入者」「一人为主」——前者太技术化，后者太口语；**主编席** 便于和并行版的 **集成主编** 区分。

### 1.2 角色分工（4 人含主编）

| 角色 | 人数 | 做什么 | 不做什么 |
|------|------|--------|----------|
| **主编席** | 1 | 唯一使用 Cursor 改 `web/`；本地跑通；推 Git；对 AI 修 bug | 不让另外 3 人同时改代码 |
| **验收席** | 1 | 对照 `index.html` 维护验收项；回归测试；录屏报 bug | 不改 `web/`、`prisma/` |
| **数据席** | 1 | 维护 `seed/` CSV（品牌、关键人、示例拜访/情报）；脱敏 | 不碰代码与环境变量 |
| **规则席** | 1 | 情报分级、拜访频率、试点名单、推广话术 | 不碰代码 |

验收席与数据席可一人兼。

### 1.3 每周节奏（30 分钟 × 2）

| 时间 | 内容 |
|------|------|
| **周一** | 主编席宣布本周 **唯一目标里程碑**；三席领辅助任务 |
| **周四** | 演示；更新本文 **§十二 验收记录**；决定是否进入下一里程碑 |

### 1.4 沟通频道

- **#sandtable-工单**：只发工单（§附录 A），主编席按序处理  
- **#sandtable-bug**：截图 + 操作步骤 + 是否必现  
- **禁止**「帮我把拜访做完」式无边界需求

## 二、产品范围（精简版）

### 2.1 本期做（MVP）

| 模块 | 功能 |
|------|------|
| 登录 | 账号密码（抄 GANT） |
| 品牌 | 列表 + 关键人手工维护 |
| 拜访 | 安排、记录、列表、简单日历、承诺条目 |
| 情报 | 录入、分级 P0/P1/P2、列表筛选 |
| 工作台 | 待办（来自承诺）+ 近期情报 |

### 2.2 本期不做（Phase 2+）

博弈推演、驾驶舱、BI 指标、自动爬虫、企微、作战手册 PDF、语音转写、GMV 追踪等。

### 2.3 原型对照

- `mod-visit` → 安排、记录、日历  
- `mod-intelligence` → 录入 + Feed + 筛选  
- 档案 → 仅品牌 + 关键人

## 三、技术路线

| 项 | 选型 |
|----|------|
| 骨架 | 复制 GANT `web/`（Next.js + Prisma + PostgreSQL） |
| 数据库 | RDS 独立库 `sandtable_prod` |
| 应用机 | 京东云 **独立主机**（不与 GANT 共机） |
| 规模 | 注册 200，峰值并发 50 |

**主编席 · Cursor 固定前缀（建议存为 Rule）：**

```text
本项目 sandtable，技术栈同 ~/Desktop/gant/web。
Next.js App Router + TypeScript + Prisma + PostgreSQL。
复用 GANT 登录与布局。当前里程碑见 docs/AI开发手册-串行主编模式.md。
```
