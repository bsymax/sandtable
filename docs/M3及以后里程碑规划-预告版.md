##### 2.3.1 佳璇 M3 需求登记 · Tab2 竞争与机会（2026-06-16）

> **来源**：佳璇 【M3 需求】· **结论**：**纳入 M3，不砍** · 与 Tab1 同「槽位升级 + 关 LLM 回退规则版」

**M2 基线（已有）**

| 项 | 现状 |
|----|------|
| 库字段 | `brand_profiles.competitive_landscape` / `growth_opportunities` 可编辑、可保存 |
| Tab2 UI | `#comp-view` / `#opp-view` + 编辑保存（`saveStrategy` → `PUT /api/brands/profile/{name_key}`） |
| 规则版 | M2 主工程为 **库内文案直出**；M3 由佳璇补 **`buildStrategyFallback()`**（全模块 context 拼段落，**非 LLM**） |

**M3 目标**

| 槽位 | DOM / 字段 | LLM ON | LLM OFF（`LLM_ENABLED=false` 或超时） |
|------|------------|--------|--------------------------------------|
| Tab1 | `#profile-ai-blurb` | Max API 一段话解读 | `buildProfileAiBlurb()`（M2 规则） |
| Tab2 | `#comp-view` / `#opp-view` | LLM 生成两字段文案 | `buildStrategyFallback()` → 填入两栏 |

**Context 范围（Prompt 输入，服务端聚合为主）**

| 模块 | 数据来源（建议 Max 在 API 内组装） |
|------|--------------------------------------|
| 经营 | `brand_metrics` 最新周 + 近几周趋势 |
| 情报 | `GET /api/intel/briefing/{name_key}` 等价数据（P0/P1、周报摘要） |
| 拜访 | 近 90 天拜访/承诺/待办、未兑现承诺 |
| 档案 | `brand_profiles` 定位/组织/潜规则 + 完整度 |

**接口约定（Max W1 交付，佳璇 W2 对接）**

```http
POST /api/brands/profile/{name_key}/ai/strategy
→ 200 {
  "competitive_landscape": "...",
  "growth_opportunities": "...",
  "source": "llm" | "fallback",
  "cached": false
}
```

- **不强制自动落库**：默认 **展示**；采销可 **编辑 → 保存**（沿用 M2 `PUT`）。可选：提供「采纳 LLM 建议」按钮写入库。
- **Prompt 规则**：输出两段中文段落，分别对应竞争格局 / 增长机会；禁止编造无 context 的数据。
- **降级**：失败时 `source=fallback`，正文与 `buildStrategyFallback()` 一致。

**M3 验收（佳璇 D-J-M3）**

| # | 项 | 标准 |

| D-J-M3-1 | Tab2 LLM ON | 换品牌两栏文案变；与 Tab1 一样标注 AI 生成 |
