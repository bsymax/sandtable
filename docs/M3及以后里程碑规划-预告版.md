## 三、M4 · 权限 + LLM 治理 + 数仓真接 + 模块稳态（~8 工作日 · ~20 人天）

> 原「M4 权限治理」与「M5 数仓稳态/模块体验」**合并为一拍**。  
> **分工原则**：Max 只做 **角色/审计/首接管道/合并**；三人 **各牵一块联调与验收**，培翛加量牵头拜访侧与数仓交叉验证。

### 3.1 目标

- 四类角色权限生效（采销 / 品类负责人 / 只读管理者 / admin）。  
- LLM 可审计、可配额、按角色开关。  
- **5 品牌首条内网 BI 真 GMV** 写入 `brand_metrics`（CSV 摆渡或 DTS 预发，见 §3.4）。  
- 数仓同步可重跑、有质量规则与告警（技术限流，非 50 人日活模型）。  
- 泰山 **预发** 环境可访问（为 M5 切流减风险）。
