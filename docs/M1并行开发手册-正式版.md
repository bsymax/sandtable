## 产品定位 · 商分能力一次做成一个产品

商分侧长期希望采销能拥有 **「品牌报告」「拜访助手」「情报体系」「谈判教练」** 等能力，但**此前并未落成独立产品**——要么停留在方案与 demo，要么只能按块零散推进（先做报表、再做拜访表、再做情报 Excel……彼此不联通）。**品牌沙盘 M1 的意义，是第一次把这四块能力按统一架构、统一品牌主数据、统一入口，一次性开发成一个可上线的产品**，而不是事后再把四个系统硬拼在一起。


| 商分能力方向（一直想做的） | 若分块做会怎样                   | 本系统模块（一次建成）                     | 侧栏入口       | M1 负责人        |
| ------------- | ------------------------- | ------------------------------- | ---------- | ------------- |
| **品牌报告**      | 单独做 BI 看板或 Excel，与档案、拜访脱节 | 品牌档案（经营看板 + 品牌简介 + 完整度）         | 品牌档案       | 佳璇            |
| **拜访助手**      | 单独做拜访表/日历，看不到品牌经营与情报      | 智能拜访助手（安排 → 记录 → 承诺 → 待办 → 健康度） | 智能拜访助手     | 培翛（**主干已交付**） |
| **情报体系**      | 单独做情报登记表，无法反哺档案与拜访提醒      | 品牌情报流（录入、分级、Feed、筛选、周报）         | 品牌情报流      | 开开            |
| **谈判教练**      | 单独做策略文档或靠老法师口传，难沉淀        | 工作台聚合 + 拜访前提醒 + 档案/拜访中的策略提示     | 今日工作台 + 联动 | Max（S5 聚合）    |


**M1 各块做到什么**：品牌报告＝5 品牌真数据、指标卡与图表；拜访助手＝MySQL 真库全链路；情报体系＝录入/列表/周报规则版；谈判教练＝**规则与静态演示级**（待办、P0 情报、健康度预警、拜访前未兑现承诺等），完整博弈推演与谈判策略模拟见设计方案 M4，放 **M2+**。

**「一站式」在 M1 的含义**：采销不再在 BI、Excel、微信纪要、零散表之间切换——一次登录进工作台（今天该干什么）→ 品牌档案（经营与人物）→ 带着提醒去拜访 → 情报与承诺挂在同一 `brands.name_key` 下可追溯。并行开发只是**内部分工**，Max 合并后对外是**一个产品、一个网址**。

**Docker 跑 MySQL（推荐，全队统一这条命令）：**

```bash
docker run -d --name sandtable-mysql \
  -e MYSQL_ROOT_PASSWORD='Sandtable@2026' \
  -p 3306:3306 \
  -v sandtable-mysql-data:/var/lib/mysql \
  mysql:8 \
  --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
```

> 统一 root 密码 `Sandtable@2026`，写进各自 `.env`，交付包就能互相直接跑（`.env` 不要发到群里截图）。

**Docker 装不上的备选：免安装版 MySQL（Max 本机实际采用此方式）**

```bash
mkdir -p ~/mysql-sandtable && cd ~/mysql-sandtable
curl -L -o mysql.tar.gz "https://cdn.mysql.com/Downloads/MySQL-8.0/mysql-8.0.46-macos15-arm64.tar.gz"
tar xzf mysql.tar.gz && ln -s mysql-8.0.46-macos15-arm64 mysql
xattr -r -d com.apple.quarantine mysql-8.0.46-macos15-arm64
mkdir data
./mysql/bin/mysqld --initialize-insecure --datadir=$HOME/mysql-sandtable/data --basedir=$HOME/mysql-sandtable/mysql
./mysql/bin/mysqld --datadir=$HOME/mysql-sandtable/data --basedir=$HOME/mysql-sandtable/mysql --port=3306 --bind-address=127.0.0.1 &
./mysql/bin/mysql -uroot --protocol=tcp -e "ALTER USER 'root'@'localhost' IDENTIFIED BY '你的密码';"
```

> ⚠️ 导入建表脚本时必须带字符集参数，否则中文变乱码：
> `mysql -uroot -p --default-character-set=utf8mb4 < database/schema.sql`
