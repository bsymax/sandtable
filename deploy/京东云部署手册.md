## 0. 架构（部署后长什么样）

```
用户浏览器
    ↓ HTTP :80
京东云 Ubuntu 24.04
├── Nginx
│   ├── /api/*  → 反代 127.0.0.1:8000
│   ├── /docs   → FastAPI Swagger
│   └── /*      → 静态文件 /opt/sandtable/web/
├── sandtable.service（uvicorn × 4 workers）
│   └── FastAPI 主工程 server/main.py
└── MySQL 8（本机 :3306，仅 localhost）
    └── 库 brand_sandtable
```

**M2 接 LLM**：仍走外部 API，服务器只加 `httpx` 调用，**不必升配**。

## 1. 你需要在京东云控制台做的（约 5 分钟）

### 1.1 确认套餐与镜像

| 项 | 值 |
|----|-----|
| 套餐 | 4核 16GB 100GB SSD **8Mbps** |
| 镜像 | **Ubuntu 24.04**（推荐 UEFI 版） |

### 1.2 获取公网 IP

控制台 → 轻量云主机 → 复制 **公网 IP**（形如 `123.56.x.x`）

### 1.3 重置 root 密码

实例详情 → 重置密码 → 设一个强密码  
⚠️ **不要把 root 密码发给 AI 或微信群**；SSH 时在本机终端输入即可。

### 1.4 防火墙放行（必做）

实例 → **防火墙** → 添加入站规则：

| 协议 | 端口 | 来源 |
|------|------|------|
| TCP | 22 | 你的办公 IP 或 0.0.0.0/0（临时） |
| TCP | **80** | 0.0.0.0/0 |
| TCP | 443 | 0.0.0.0/0（预留 HTTPS） |

> **80 未放行 = 外网打不开网站**，部署成功也白搭。

## 2. 一键部署（推荐 · 在你 Mac 上执行）

项目根目录打开终端：

```bash
cd ~/Desktop/sandtable
bash deploy/pack-and-upload.sh <你的公网IP>
```

脚本会：

1. 打包 `server/` + `web/` + `database/`（不含 `.env`；Mac 清 xattr）
2. `scp` 上传到云主机（**2 次**密码：tar 包 + 部署脚本）
3. **`nohup` 后台**执行 `sandtable-main-deploy.sh`（**1 次**密码；SSH 断开也会继续）
4. 本机 **curl 轮询** `/api/brands`，约 5～10 分钟出结果（无需反复输密码）

> ⚠️ 不要在中途关终端；若已显示 `DEPLOY_STARTED`，等脚本自己跑完或见 §9 查日志。

## 3. 部署脚本说明（复盘用）

| 文件 | 作用 |
|------|------|
| `deploy/pack-and-upload.sh` | **本机执行**：打包 + 上传 + 触发远程部署 |
| `deploy/sandtable-main-deploy.sh` | **云主机执行**：装依赖、MySQL、Nginx、systemd |
| `deploy/smoke-test.sh` | 部署后 HTTP 冒烟 |

### 远程脚本 8 步在做什么

| 步骤 | 内容 |
|------|------|
| 1 | 若未装依赖则 `apt`（**已装则跳过**，避免重启 ssh）；`NEEDRESTART_MODE=l` |
| 2 | MySQL：建库/用户；密码持久化 `/root/.sandtable-db-credentials`；连不上则自动 recover |
| 3 | 解压到 `/opt/sandtable`；**chmod 644 web/**（防 Nginx 403） |
| 4 | 前端去掉 `http://127.0.0.1:8000`，走同源 `/api` |
| 5 | Python venv + `pip install -r requirements.txt` |
| 6 | 导入 `database/schema.sql` + ORM 补表 |
| 7 | systemd `sandtable.service`（4 workers）+ Nginx 反代 |
| 8 | ufw 放行 22/80/443 |

### 部署后关键路径

| 路径 | 说明 |
|------|------|
| `/opt/sandtable/` | 项目根 |
| `/opt/sandtable/server/.env` | 数据库密码（chmod 600） |
| `/opt/sandtable/DEPLOY-CREDENTIALS.txt` | 当次部署密码汇总（**仅服务器本机**） |
| `/root/.sandtable-db-credentials` | **重复部署复用**的 MySQL 密码（chmod 600） |
| `/root/sandtable-deploy.log` | 后台部署日志（`tail -f` 排查用） |
| `/etc/nginx/sites-available/sandtable` | Nginx 配置 |
| `/etc/systemd/system/sandtable.service` | 后端守护 |

## 4. 日常运维命令（SSH 登录后）

```bash
# 服务状态
systemctl status sandtable nginx mysql --no-pager

# 重启后端（改代码后）
systemctl restart sandtable

# 看后端日志
journalctl -u sandtable -f

# 看 Nginx 访问日志
tail -f /var/log/nginx/access.log

# 端口监听
ss -tlnp | grep -E '80|8000|3306'
```

## 6. 故障排查

| 现象 | 检查 |
|------|------|
| 外网打不开 | 京东云防火墙 **80**；`ufw status` |
| 502 Bad Gateway | `systemctl status sandtable`；`journalctl -u sandtable -n 30` |
| 页面开但无数据 | MySQL 是否 active；`.env` 密码 |
| API 500 | `journalctl -u sandtable -n 50`；schema 是否导入 |
| 中文乱码 | 导入 schema 时应用 utf8mb4 |
| **visit.html 403** | `ls -l /opt/sandtable/web/visit.html` 非 644 → `chmod -R a+rX /opt/sandtable/web` |
| **Connection closed by remote host** | apt 重启 ssh；用新版 `pack-and-upload`（nohup）；查 `/root/sandtable-deploy.log` |
| **MySQL ERROR 1045** | 重复部署密码不一致；脚本已持久化 `/root/.sandtable-db-credentials` 或自动 recover |
| **tar xattr 告警** | Mac 打包无害；可忽略 |
| 只有 Nginx 欢迎页 | 部署未完成；`tail -50 /root/sandtable-deploy.log` 后重跑 pack-and-upload |

### 8.3 踩坑与修复（按发生顺序）

| # | 现象 | 原因 | 修复 |
|---|------|------|------|
| 1 | Step 4 `sed: unknown option` | `visit.html` 一行含 `:`/`?`，原 sed 替换写法非法 | 改为 `sed 's\|http://127.0.0.1:8000\|\|g'` |
| 2 | Step 2 `ERROR 1045` 第二次部署 | 首次随机 MySQL root 密码未保存，第二次又生成新密码 | 密码写入 `/root/.sandtable-db-credentials`；连不上则 skip-grant-tables recover |
| 3 | `Connection closed by remote host` | `apt-get install` 触发 **ssh.service 重启**，前台 SSH 部署被掐断 | 改 **nohup 后台** + 依赖已装则 **跳过 apt** + `NEEDRESTART_MODE=l` |
| 4 | 大量 `tar: LIBARCHIVE.xattr...` | Mac 扩展属性 | **无害**；打包 `xattr -cr` + `COPYFILE_DISABLE=1` |
| 5 | 外网只有 Nginx 欢迎页 | 第 3 次部署中断，sandtable 站点未配完 | 第 4 次 nohup 跑完 |
| 6 | `/visit.html` **403** | 本地 `visit.html`、`api.js` 权限 **600**，Nginx(www-data) 不可读 | 服务器 `chmod -R a+rX /opt/sandtable/web`；脚本 Step 3 自动 chmod；本地改 644 |
