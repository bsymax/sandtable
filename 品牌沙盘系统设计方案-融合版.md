# 品牌沙盘系统设计方案

> 融合版 V1.0 | 2026年6月 | 密级：内部  
> 融合思路：开开的竞对视角定战略高度 × 佳璇的闭环设计搭系统骨架 × 培翛的覆盖广度填血肉

---

## 一、为什么要做品牌沙盘（战略定位）

### 1.1 京东采销的困局

一个采销同时管理多个品牌，每个品牌的情况散落在不同系统中——销售数据在BI报表里，历史沟通在个人微信/邮件里，品牌方人员变动在脑子里，竞品动态需要自己手动搜集。当需要与品牌方进行一次关键谈判时，采销往往花大量时间"拼图"，最终判断高度依赖个人经验。

更深层的痛点是**博弈不对等**。品牌方对自己的成本结构、产品规划、渠道策略了如指掌，而京东采销往往只能看到品牌在京东平台上的表现，缺乏对品牌全局战略的理解。采销在谈判中处于信息劣势。

### 1.2 竞对正在建立AI壁垒

这不是一个孤立的内部效率问题。外部竞争格局正在加速变化：

- **美团牵牛花Claw** 已具备"AI策略库+自动化管理"能力
- **拼多多黑标体系** 在产业带品牌争夺中形成吸引力
- **抖音用大模型佣金分级** 引导品牌预算向抖音倾斜

京东若不在品牌关系管理上建立AI壁垒，将在品牌资源博弈中逐渐丧失信息优势。品牌沙盘不只是采销的效率工具，而是**京东在品牌关系管理上的战略防御武器**。

### 1.3 产品定位

品牌沙盘是一个AI驱动的品牌管理与作战指挥平台。它不只是信息的聚合展示。

- **情报中枢**：汇聚品牌内外部全维度信息，形成"活的品牌档案"
- **博弈引擎**：基于多角色模拟，推演品牌方行为并输出采销策略建议
- **行动系统**：将策略结论转化为可追踪的待办任务，闭环管理
- **组织记忆**：沉淀每一次品牌互动的经验和结论，让组织能力不随人员流动而流失
- **双向武器**：既能对内推演博弈，也能对外生成面向品牌方的价值论证报告

**产品Slogan**：看清品牌全局，预演对手下一步，让每一次拜访都带着策略出发。

### 1.4 北极星指标

品牌资源获取转化率——品牌沙盘上线后，采销通过系统建议达成的品牌资源合作（新品首发、包销大单、广告投入增加）数量，环比提升 ≥ 20%。

## 二、核心用户与使用路径

### 2.1 用户痛点矩阵

| 痛点维度 | 一线采销 | 品类负责人 | 事业部管理者 |
|---------|---------|-----------|------------|
| 信息获取 | 需登录5+系统才能凑齐品牌完整画像 | 无法快速横向对比各采销的工作质量 | 不知道哪个品牌关系在恶化 |
| 经验传承 | 老采销离职带走品牌关系和博弈经验 | 新人上手周期3-6个月 | 组织能力无法沉淀 |
| 决策支持 | 谈判前靠"老法师"口传心授 | 资源分配依赖述职PPT | 战略决策缺乏推演工具 |
| 执行追踪 | 聊完就完了，没有系统记录和跟进 | 不知道下属待办是否在推进 | 重要事项容易遗漏 |

### 2.2 系统使用路径（"感知→判断→行动→反馈→学习"闭环）

采销打开品牌沙盘，进入**"今日工作台"**：

1. **感知**：数据异动预警 + 情报推送 → "A品牌GMV连续两周跌15%，竞对在抖音上了独家新品"
2. **判断**：AI博弈推演 → "如果我是A品牌决策者，我现在核心诉求是保住抖音增长，京东的议价窗口在收缩。建议本周内拜访，核心筹码是京东的用户画像优势"
3. **行动**：生成拜访作战手册 + 一键转为待办，@责任人
4. **反馈**：拜访后AI从纪要中提取承诺和结论，更新品牌档案，追踪品牌经营数据是否改善
5. **学习**：成功的博弈策略沉淀进案例库，失败的推演标记归因，AI模型持续优化

管理者视角则是：
- 全局战局dashboard（横向俯瞰所有品牌竞争位势）
- 团队待办完成率与超期预警（纵向管理执行）
- 品牌健康度红灯自动升级（异常管理）

## 四、各模块核心设计

### M1 品牌全景档案——"活的画像"

档案不是静态页面，而是随着数据更新、情报汇入、采销互动持续演进的。

**六大信息域：**

| 信息域 | 核心内容 | 更新机制 |
|-------|---------|---------|
| 经营数据 | GMV/毛利/转化率/退货率/市占，支持同比/环比/下钻到SKU | 系统自动同步 |
| 品牌方画像 | 组织架构、决策链（谁能拍板）、产品线、策略方向、核心KPI | 采销维护 + 建联记录AI提取 |
| 历史互动 | 沟通时间线、承诺与履约追踪、关系温度评分 | AI从拜访纪要自动提取 |
| 重大事件 | 品牌方（CEO变更/融资/质量事故）、行业（政策变化、竞品动作）、京东侧（合同到期等） | 情报系统自动采集 + 采销补充 |
| 竞对关系 | 同品类竞对列表、优劣势对比、品牌间博弈历史 | 系统自动关联 |
| 渠道版图 | 品牌在JD/TM/DY/PDD/秒送/线下/下沉的销售占比与增速 | 数据系统同步 |

**档案健康度评分**：自动计算数据完备度（0-100分），标记缺失或过期字段（如人员信息超过90天未更新），提醒负责采销补充。

### M2 品牌情报流——"全源情报，分级推送"

**情报分类与分级：**

| 级别 | 定义 | 推送方式 | 示例 |
|-----|------|---------|------|
| P0-紧急 | 24小时内需响应 | 企微+邮件强提醒 + 自动建待办 | 品牌CEO更换、重大质量事故 |
| P1-重要 | 本周内关注行动 | 情报周报置顶 + 待办建议 | 销售连续下滑、竞对独家合作 |
| P2-常规 | 知道即可 | 周报中呈现 | 新品发布、常规人事变动 |
| P3-背景 | 辅助积累 | 不推送，可检索 | 行业报告、品牌历史 |

**情报周报**：每周一自动生成，包含AI一句话总结、关键数据异动、本周商家互动摘要、待办完成与下周建议、竞对动态简报。支持按品牌/品类/角色订阅推送。

**跨品牌关联**：AI自动识别不同品牌情报之间的关联（如"A品牌降价可能是在应对B品牌的新品发布"），在周报中体现。

### M3 多维预警系统——"不只盯数据，盯关系"

| 预警类型 | 触发条件示例 | 预警逻辑 |
|---------|------------|---------|
| 经营数据异动 | GMV连续2周跌幅超15%、转化率异常下降 | 时序异常检测 + AI归因 |
| 关系温度预警 | 关键人员离职、超过N天无建联、承诺兑现率下降 | 关系健康度评分模型 |
| 竞对威胁预警 | 品牌在竞对平台增速显著高于京东、竞对给品牌独家政策 | 跨平台数据对比 |
| 供给恶化预警 | 品牌在京东SKU数持续减少、核心品缺货超X天 | 供给趋势监控 |
| 合约风险预警 | 合同到期前N天、返利条款未达标 | 合同系统联动 |

**预警→拜访联动**：触发预警后，自动推荐"建议拜访"并附带拜访要点简报。不是每条预警都建议拜访——系统根据预警类型和严重程度做初筛。

### M4 博弈推演引擎——"模拟品牌方在想什么"

**核心场景：**

- **单品牌策略推演**：基于品牌经营数据+历史行为模式+行业趋势，推演品牌方接下来可能采取的动作及概率
- **品牌间博弈推演**：模拟品类内主要玩家的竞争互动，识别"鹬蚌相争"机会——例如用Top2逼迫Top1开放独家款
- **谈判策略模拟器**：采销输入假设动作（"要求降价5%""要求独家首发"），系统推演品牌方可能的反应及概率，给出最优谈判区间和让步梯度
- **竞对平台冲击推演**：评估品牌在竞对平台的新动作对京东的风险，输出挽留/进攻优先级

**推演输出标准报告：**
1. 推演假设（前提条件和假设）
2. 品牌方视角分析（如果我是XX品牌决策者，核心诉求是什么，筹码是什么，担心什么）
3. 情景推演（乐观/基准/悲观三种情景的概率分布）
4. 策略建议（谈判目标、筹码排序、关键话术、替代方案）
5. 风险提示（意外情况和应对预案）

**关键原则：**
- **置信度标注**：每条推演附带高/中/低置信度，基于数据完整度+行为可预测性+信息新鲜度。低置信度自动触发补充情报采集。
- **A/B方案**：关键博弈节点输出进攻型和防守型两套方案，标注预期收益和风险。
- **可追溯**：每条建议可点击查看背后的支撑数据和推理链路。
- **可修正**：采销可对推演结果做"有用/无用"评价，反馈纳入模型优化。低分推演自动进入人工复核。
- **有边界**：AI明确说明"以下是我的分析，最终决策由你做出"。

### M5 品类战局视图——"一眼看穿品类竞争格局"

- **品牌竞争矩阵**：横轴JD市占 × 纵轴行业增速 × 气泡GMV绝对值，四象限分层（明星/金牛/机会/问题），不同象限对应不同策略
- **品牌健康度雷达图**：每个品牌在销售表现、利润贡献、新品配合度、广告投入、独家合作深度、人员关系紧密度六个维度评分
- **品牌关系图谱**：可视化品牌间竞争/互补/替代关系，标注竞争强度
- **品牌势力变迁**：时间轴展示各品牌份额升降、排名变化，识别上升期和下滑期品牌
- **跨品牌协同识别**：AI自动识别资源置换机会——如品牌A有包销意愿但缺广告预算，品牌B有广告预算但需要独家品，建议采销联动谈判

### M6 智能拜访助手——"每一次拜访都带着策略出发"

**拜访触发机制：**

- 数据异动触发（预警系统联动）
- 时间节奏触发（距上次拜访超过约定周期、合同到期前、大促前）
- 事件触发（品牌方重大事件、行业政策变化）
- 待办驱动（上次拜访待办全部完成或长期未推进）

**拜访作战手册**（拜访前自动生成）：

| 模块 | 内容 |
|-----|------|
| 拜访背景 | 为什么去、上次拜访回顾、期间关键变化 |
| 数据洞察 | 品牌在京东核心数据变化、行业对比、需重点讨论的数字 |
| 谈判目标 | 本次核心诉求、优先级排序、期望达成的具体成果 |
| 博弈策略 | 基于推演引擎输出的谈判策略和话术 |
| 资源弹药 | 京东能给对方什么（流量资源、数据赋能、联合营销等筹码） |
| 风险预案 | 品牌方可能提出的诉求及应对方案 |
| 待办预置 | 拜访后可能需要跟进的待办模板 |

**拜访后闭环**：AI从纪要中提取讨论议题、双方承诺、后续待办、关系温度 → 自动更新品牌档案互动时间线 → 未完成承诺进入待办追踪 → 追踪拜访后N天内品牌经营数据变化评估拜访成效。

**拜访节奏管理**：为每个品牌设定拜访基线（KA每月1次，成长品牌每季度1次），生成季度拜访日历，临近计划未执行主动提醒。

### M7 待办指挥中心——"结论必须落到人"

**多源自动聚合**：从拜访纪要、博弈推演建议、数据异动、供给缺口、管理者指令等场景自动生成待办。

**待办结构化**：每条待办包含责任人、截止时间、关联品牌、优先级、来源（可点击跳转到来源上下文）、完成标准（避免"做了但等于没做"）、阻塞标记。

**升级机制**：

| 时间 | 动作 |
|-----|------|
| 到期前48h | 提醒责任人 |
| 到期前24h | 再次提醒 |
| 逾期第1天 | 通知责任人 + 标记橙色 |
| 逾期第3天 | 抄送直属上级 |
| 逾期第7天 | 标红进入管理者关注列表 |

**管理者全局舱**：团队待办总量、完成率趋势、超期率，按采销/品牌/优先级维度下钻。待办积压联动品牌健康度——同一品牌多待办超期，品牌档案中风险指数自动上升。

### M8 全渠道供给分析——"别人有京东没有，就是机会"

- **SKU级匹配**：逐一对比品牌在JD vs TM/DY/PDD的SKU差异
- **缺口优先级排序**：P0高价值缺口（竞对月销>X万且增速>Y%）、P1战略缺口（品牌核心产品线缺失）、P2常规缺口、P3长尾缺口
- **自动转待办**：每个缺口SKU生成补品待办，AI推荐合适价格和运营策略，@品牌负责采销
- **补品进度看板**：管理者按品类/品牌查看缺口总量、已补量、谈判中量、品牌拒绝量及原因

### M9 品牌资源争取工具包——"双向武器"

**对内**（采销自己看）：品类趋势+新品机会论证、竞对投放效率对比、品牌差评归因与改善建议

**对外**（给品牌方看）：
- 一键生成"京东合作价值报告"（PDF/PPT），展示品牌在京东的增长趋势、用户画像优势、投放效率 vs 竞对平台
- 品类趋势洞察推送：引导品牌将新品首发放在京东
- 竞品对标刺激：向品牌A展示竞品B在京东的广告投入和效果，激励加大投放

### M10 品牌知识库——"对抗人走知识走"

**知识类型**：品牌基础信息、产品知识、行业知识、合作历史、沟通偏好、谈判底线与让步模式

**更新机制**：
- 系统自动同步（经营数据、行业数据）
- AI从拜访纪要自动提取新知识并建议入库
- 采销手动补充/修正
- 过期提醒（超过设定周期未更新打"可能过期"标签）
- 知识冲突检测（多人维护同一品牌时AI检测冲突并提示协调）

**知识质量评分**：AI自动评估每条知识的完整性和时效性

### M11 品牌故事库——"战果沉淀，以战养战"

每个故事包含：品牌背景（数据+问题）→ 采销洞察 → 行动过程 → 遇到的阻力 → 破局关键决策 → 量化成果 → 可复用方法论和教训

**使用场景**：
- 内部培训：新人学习"京东采销怎么跟品牌做生意"
- 对外说服：跟类似品牌谈合作时作为"战果背书"
- 团队激励：优秀案例展示
- 方法沉淀：多个故事的共性提炼为品牌合作方法论

**效果追踪**：记录每个故事被下载/引用的次数，以及引用后是否促成新合作，按转化率排序让高价值故事自动置顶。

### M12 全局管理驾驶舱

一屏看全局：品牌健康度红黄绿灯、待办总览（团队完成率/超期率/按人分布）、预警中心（所有触发预警的事项一览）、供给健康度、资源争取看板（新品/包销/广告费用的争取量汇总）、人效分析（每个采销的品牌数/拜访频次/待办完成率/GMV变化）。

管理者可自定义预警规则——不同品类/阶段的品牌预警标准不同。

### M13 采销个人工作台

聚合今日待办、今日需关注的异动、今日建议联系的品牌、今日拜访安排。周报/月报自动生成草稿，减少手工汇报成本。采销个人KPI（GMV达成、品牌引入数、拜访完成率）与系统数据自动关联。

### M14 品牌分级管理体系

按战略价值/收入贡献/增长潜力/合作紧密度将品牌分为S/A/B/C四级：

| 级别 | 功能深度 | 拜访频率基线 | 预警敏感度 |
|-----|---------|------------|-----------|
| S-战略 | 全部14个模块 | 每月≥1次 | 高敏感 |
| A-核心 | 10个核心模块 | 每季度≥1次 | 中敏感 |
| B-常规 | 基础档案+预警+待办+周报 | 每半年≥1次 | 低敏感 |
| C-长尾 | 基础档案+异动提醒 | 按需 | 仅重大异常 |

分级不是固定的——品牌表现变化触发级别调整建议。

## 五、系统骨干："感知→判断→行动→反馈→学习"如何串联

五个环节不是各模块的简单堆叠，而是数据流和行为链的闭环：

```
预警系统(M3)触发异动
       ↓
情报流(M2) 补充上下文 → AI一句话总结"发生了什么"
       ↓
博弈引擎(M4) 做推演 → 输出"品牌方可能怎么反应，你应该怎么做"
       ↓
拜访助手(M6) 生成作战手册 → 或 直接生成待办(M7) @责任人
       ↓
采销执行后 → 拜访纪要回灌到品牌档案(M1) 更新互动时间线
       ↓
拜访效果评估 → 品牌经营数据变化反馈到预警系统(M3) 调整下次预警阈值
       ↓
成功案例 → 自动归档品牌故事库(M11) + 博弈策略入知识库(M10)
失败案例 → 标记归因，进入案例库供后续推演参考
```

这意味着系统会越用越聪明：第一次预警可能误报多，但随着"预警→拜访→结果"的闭环数据积累，预警精度会提升。第一次推演可能概率不准，但随着采销的"有用/无用"反馈积累，品牌方的行为模型会越来越准。

## 六、AI能力矩阵

| AI能力 | 应用场景 | 方案思路 |
|--------|---------|---------|
| 智能摘要 | 拜访纪要摘要、情报周报总结、长篇报告关键提取 | LLM文本摘要 + 结构化输出 |
| 异动检测+归因 | 品牌经营数据异常自动发现和原因分析 | 时序异常检测 + LLM自然语言解释 |
| 多角色博弈推演 | 模拟品牌方决策者、京东采销顾问的角色推演 | Multi-Agent LLM + 品牌知识库RAG + 历史案例few-shot |
| 知识图谱推理 | 品牌间关系、人员关系、产品线关系结构化表示 | 知识图谱 + 图推理 |
| 智能推荐 | 供给缺口优先级、拜访时机、谈判筹码排序 | 排序模型 + 规则引擎 |
| 自然语言查询 | 采销问"A品牌最近怎么了"直接获得答案 | LLM + RAG检索增强生成 |
| 文档生成 | 一键品牌提案、拜访手册、周报 | LLM模板化生成 |
| 意图识别+情感分析 | 从沟通记录中提取品牌方真实诉求和情绪 | LLM情感分析 + 意图分类 |

**AI输出四条原则：**
- **可追溯**——每条结论能追溯到支撑数据和推理过程，不下"黑盒结论"
- **可修正**——采销可对AI输出纠正反馈，反馈纳入模型优化
- **有概率**——预测类输出附带置信度
- **有边界**——AI明确告知"以下是我的分析，最终决策由你做出"

## 七、实施路线图

### Phase 1 · 地基期（6-8周）

**目标**：让采销能在一个地方看到品牌全景，收到异动预警，形成基础的"看→警→动"闭环。

**交付**：
- M1品牌全景档案（先做经营数据面板 + 基础品牌画像，历史互动先手动录入）
- M3预警系统（基于GMV/市占等核心指标异动检测+企微通知）
- M6拜访助手（基础版：拜访触发+作战手册生成）
- M7待办系统（基础版：创建+分配+提醒+完成标记）
- M2情报周报（5个试点品牌，覆盖厨小/卫浴）
- M14品牌分级（核心品牌打标，决定试点范围）

**验证指标**：10个采销试点，品牌档案周活跃率>60%，异动预警→拜访转化率>50%

### Phase 2 · 智能期（8-12周）

**目标**：AI能力上线，从"信息工具"升级为"决策工具"。

**交付**：
- M4博弈推演引擎（先支持单品牌推演+谈判策略模拟两个场景）
- M7待办系统完整版（升级机制+管理者全局舱）
- M2情报流完整版（全源采集+分级推送+跨品牌关联）
- M6拜访助手升级版（纪要AI解析+拜访效果追踪）
- M8全渠道供给分析（基础版：JD vs TM/DY SKU对比）
- M5品类战局视图（品牌竞争矩阵+关系图谱）

**验证指标**：周生成博弈推演报告>50份，建议采纳率>60%，待办闭环率>80%，周报自动生成覆盖率>90%

### Phase 3 · 协同期（8-12周）

**目标**：品类全局视角+跨品牌协同+组织记忆沉淀+管理者驾驶舱。

**交付**：
- M5战局视图完整版（品牌势力变迁+跨品牌协同识别+资源置换建议）
- M8供给分析完整版（缺口优先级排序+补品追踪看板）
- M10品牌知识库（结构化管理+AI自动提取+过期提醒+冲突检测）
- M11品牌故事库（模板化沉淀+效果追踪）
- M12全局管理驾驶舱
- M13采销个人工作台
- 品牌间博弈推演（M4场景扩展）

**验证指标**：管理者周活跃率>80%，供给缺口补充率>30%，品牌档案健康度>85%

### Phase 4 · 深化期（持续迭代）

- M9品牌资源争取工具包（一键提案、品牌方视角报告）
- 移动端轻量版（拜访场景快速操作）
- 经销商/代销库存监控接入
- AI推演模型基于采销反馈持续迭代
- 品牌方人员变动追踪
- 绩效系统打通

## 八、关键风险与应对

| 风险 | 影响 | 应对 |
|-----|------|------|
| 采销使用习惯难改变 | 做好了没人用 | 从"被动推送"切入（情报周报、异动企微提醒），嵌入现有工作流而非要求开新系统；今日工作台降低第一屏操作成本 |
| AI推演准确性问题 | 采销信任崩塌 | AI建议定位为"参考"而非"决策"，始终标注置信度+数据源+推理链，建立"有用/无用"反馈闭环持续优化 |
| 知识库冷启动 | 初期品牌档案空空如也 | 设计贡献激励机制（贡献者榜单+节省了多少时间反馈），AI自动从已有数据源提取填充 |
| 数据质量不足 | AI分析不可信 | Phase 1优先做数据校验和异常标记，核心功能不依赖外部数据 |
| 外部数据源不稳定 | 全网对比等功能无法运行 | 核心路径（档案/待办/预警）不依赖外部数据，外部数据做锦上添花 |
| 品牌方信息敏感 | 关系风险 | 对内使用不外传，品牌提案类输出设权限控制，敏感信息按角色分级可见 |
